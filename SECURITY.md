# Security Policy

## Reporting A Security Issue

If you find a security-sensitive issue, please do not publish exploit details in a public issue first.

Open a minimal private report through the repository hosting platform if available, or contact the maintainer through the project's published contact channel. Include:

- Affected version or commit.
- Steps to reproduce.
- Impact.
- Suggested fix if known.

## Scope

Relevant security issues include:

- Unsafe file handling.
- Path traversal or unintended file overwrite.
- Sensitive data leakage.
- Permission misuse.
- Crashes caused by malformed files when they create a security risk.

## Out Of Scope

- Requests for copyrighted music files.
- Requests to bypass platform accounts, payments, or access controls.
- Reports that require illegal access to third-party systems.

## Privacy

The app is designed to process files locally on the device and does not actively upload file contents.
